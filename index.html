SITUACIÓN PROBLEMA

OBJETIVO GENERAL

Desarrollar el Sistema automatizado Gestor de Empleados y Contrataciones para la empresa CentroVenezolano C.A

OBJETIVOS ESPECIFICOS

  1. Analizar los procesos de contratación y asignación de permisos que se realizan dentro de la organización.
  2. Diseñar el módulo de empleados y contratos.
  3. Desarrollar las funcionalidades del módulo de empleados y contratos.
  4. Realizar pruebas de funcionamiento para el modulo propuesto.
  5. Implantar el modulo propuesto en la empresa.

CICLO DE VIDA DE XP

EXPLORACIÓN

En esta fase los clientes realizan las historias de usuario y los desarrolladores definen las herramientas a utilizar

PLANIFICACIÓN

Durante esta fase se refinan, ordenan y clasifican las historias de usuarios realizadas por el cliente

ITERACIÓN

En esta fase se decide el numero de iteraciones necesarias para culminar el desarrollo del sistema

PRODUCCIÓN

En esta fase el producto sale de un ambiente de desarrollo a uno de produccion.

MANTENIMIENTO

Durante esta fase se pueden agregar nuevas funcionalidades al sistema

MUERTE

Esta fase define la culminacion del proyecto

NUEVO PERMISO

FACILIDADES

  • Al momento de instalar las aplicaciones en los servidores, ya todas las dependencias necesarias para su ejecución estaban instaladas.
  • Interacción constante con el personal administrativo de la organización, permitiendo una constante retroalimentación del proyecto.
  • Se generó una buena comunicación con el tutor industrial y el tutor académico lo que facilito el desarrollo del sistema.
  • El uso de una plantilla HTML facilito las cosas a la hora de realizar el diseño de la interfaz del sistema.

DIFICULTADES

  • La implementación del framework AngularJS, situación que fue solventada buscando información y realizando pruebas.

APORTES

  • Con la implementación del sistema se logró reducir las ocurrencias de errores a la hora de registrar información referente a los empleados.
  • El sistema permitió reducir el tiempo adicional que utilizaba el personal administrativo a la hora de buscar información referente a un empleado en particular.
  • Gracias a la implementación del sistema la gerencia de la empresa tiene un mayor control sobre las actividades que realiza cada uno de sus empleados.

CONOCIMIENTOS TEÓRICOS

  • Beneficios de la utilización de frameworks, los cuales permiten el desarrollo rápido de aplicaciones y la reutilización de componentes de software.
  • Beneficios de la implementación de metodologías de desarrollo ágil como XP, las cuales permiten una menor taza de errores y las cuales tienen la capacidad de adaptarse al verdadero ciclo de vida del desarrollo del software el cual es cambiante en cualquier fase del mismo.

CONOCIMIENTOS PRÁCTICOS

  • Experiencia en lenguajes de programación como JavaScript, HTML5, CSS3.
  • Dominio de Sailsjs (frameworkpara Nodejs).
  • Dominio de Angularjs (framework para javascript del lado del cliente).
  • Manipulación de sockets para aplicaciones en tiempo real
  • Experiencia en el trato con los clientes.

RECOMENDACIONES

Por motivos de tiempo, el modelo entidad relación fue normalizado hasta su tercera forma normal, sin embargo es recomendable llevarlo por lo menos a una quinta forma normal para evitar aún más la redundancia de datos.

RECOMENDACIONES

Debido a que la ejecución del sistema es completamente en local no se hizo uso de zonas desmilitarizadas ni de firewalls, sin embargo si el sistema quiere ser adaptado a un ambiente con acceso del mundo exterior es recomendable el uso de zonas desmilitarizadas y un mayor control en la parte de seguridad para acceder al sistema.

RECOMENDACIONES

Debido a que la ejecución del sistema es completamente en local las validaciones para la conexión de los sockets, no se hicieron, esto debido a que la gerencia de sistemas no hizo hincapié en realizarlas, sin embargo si se quiere dar acceso del mundo exterior es recomendable agregar un mecanismo de seguridad como la autenticación por token para validar las conexiones de los sockets.
