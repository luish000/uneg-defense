SITUACIÓN PROBLEMA

OBJETIVO GENERAL

Desarrollar el Sistema Gestor de Empleados y Contrataciones para la empresa CentroVenezolano C.A

OBJETIVOS ESPECIFICOS

  1. Analizar los procesos de contratación y asignación de permisos que se realizan dentro de la organización.
  2. Seleccionar las tecnologías de desarrollo a utilizar.
  3. Levantar la información referente a los requerimientos del sistema.
  4. Planificar iteraciones con entregas del sistema.
  5. Desarrollar el sistema de acuerdo a la planificación realizada.
  6. Realizar las pruebas para la depuración del sistema.
  7. Documentar el sistema.

METODOLOGÍA Y TECNICAS APLICADAS

PROGRAMACIÓN EXTREMA (XP)

ROLES EN XP

  • Programador
  • Cliente
  • Tester
  • Tracker
  • Entrenador
  • Consultor
  • Gestor

CICLO DE VIDA DE XP

EXPLORACIÓN

En esta fase los clientes realizan las historias de usuario y los desarrolladores definen las herramientas a utilizar

PLANIFICACIÓN

Durante esta fase se refinan, ordenan y clasifican las historias de usuarios realizadas por el cliente

ITERACIÓN

En esta fase se decide el numero de iteraciones necesarias para culminar el desarrollo del sistema

PRODUCCIÓN

En esta fase el producto sale de un ambiente de desarrollo a uno de produccion.

MANTENIMIENTO

Durante esta fase se pueden agregar nuevas funcionalidades al sistema

MUERTE

Esta fase define la culminacion del proyecto

IMPLEMENTACIÓN DE LA METODOLOGÍA

ROLES

  • Programador: Pasante
  • Cliente: Usuario final
  • Tester: Gerente de sistemas y vicepresidente de CentroVenezolano C.A.
  • Entrenador y Tracker: Gerente de sistemas.
  • Consultor: Analistas de sistemas.
  • Gestor: Vicepresidente.

FASE DE EXPLORACIÓN

Durante esta fase se realizaron las historias de usuario y se definieron las herramientas a utilizar para el desarrollo

CARACTERÍSTICAS DEL SISTEMA

  • Registro de usuarios con roles
  • Actualización de roles de usuarios.
  • Eliminar usuarios registrados
  • Registrar, editar y eliminar empleados.
  • Ver perfil de empleado.
  • Listar empleados activos e inactivos de la empresa.
  • Generar y actualizar contratos por empleado.
  • Gestionar vacaciones y permisos de los empleados.
  • Gestion de notificaciones.
  • Comunicación en tiempo real de eventos.
  • Registro de eventos en el sistema.

ARQUITECTURA DEL SISTEMA

Tras reuniones con el gerente de sistemas se definio que se trabajaria bajo una arquitectura cliente - servidor

HERRAMIENTAS

Para el desarrollo del backend se tomo la decision de utilizar sailsjs y para el desarrollo del frontend utilizar angularjs

FASE DE PLANIFICACIÓN

En esta fase se refinaron, organizaron y priorizaron las historias de usuario construidas por el cliente
